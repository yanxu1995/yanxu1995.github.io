コンピューター構成要素

プロセッサ

コンピューターの種類

一般的なコンピューターは、五つの装置で構成されている

  • 制御装置
  • 演算装置
  • 記憶装置
  • 入力装置
  • 出力装置

処理装置は、演算装置や制御装置を持ち、一般的にCPUや、プロセッサと呼ばれている。（Central Processing Unit / Processing Unit）

このような基本構造の大本は、ノイマン型コンピューターとよばれ、二つの大きな特徴がある。

  1. プログラム内蔵方式（Stored Program）:あらかじめプログラムとデータを主記憶装置に格納しておく方式。
  2. 逐次制御方式（Sequential Control）：命令を主記憶装置から取り出して解読し、その内容によってデータを取り出し、処理を実行するという一連の流れを自動的に繰り返すことで、プログラムを実行していく方式。

コンピューターにはPC（パーソナルコンピューター）、WS（ワークステーション）、スーパーコンピューターと汎用コンピューター（メインフレーム）などがある。

CPU

CPUは、コンピューターの中心として演算や制御を行う装置である。
CPUのアーキテクチャは、大きく二つに分けられる。

  • CISC（Complex Instruction Set Computer）
  • RISC（Reduced Instruction Set Computer）
CISC RISC
命令セット 簡易+高機能 単純で基本
命令長 命令ごとに異なり、長い 固定長で短い
レジスタ数 少ない 多い
命令実行方式 マイクロプログラム制御方式 ワイアードロジック制御方式
パイプライン処理 適さない 適する
同一処理あたりのステップ数 少ない 多い

CPUの構成と動作の仕込み
CPUは制御装置と演算装置から構成され、それぞれ高速で容量が小さいレジスタを持ち命令解読、演算を行う。

レジスタは、その使用目的によって制御用レジスタと演算用レジスタに分けられる。

  • 制御用レジスタ

    • 命令アドレスレジスタ（IAR：instruction address register）
    • 命令レジスタ（IR：instruction register）

  • 演算用レジスタ

    • アキュムレータ
    • インデックスレジスタ

  • 汎用レジスタ

主記憶装置上のプログラムは、命令サイクルと実行サイクルを繰り返すことで処理を進めていく。

  • 命令サイクル

    • CPUは、命令アドレスレジスタに格納されている番地を主記憶総©ひぃへ送るとともに、読み出し信号を読み書き制御部に伝える。
    • 主記憶装置は、指定された番地から命令を読み出し、CPU内の命令レジスタに格納する。
    • 一つの命令の取り出しが終了すると、プログラムレジスタの値を次に実行すべき命令のアドレスに変更する。通常命令の場合は、プログラムレジスタに命令乗をアドレスに加算する。

  • 実行サイクル

    • ２で命令レジスタに格納された命令のうち「オペレータ部（命令コード）」を命令解読器で解読する。
    • 命令解読の結果に基づいて、「オペランド部（アドレス情報）」をアドレス変換機構へ送り、有効アドレスの計算を行う。
    • 有効アドレスに基づいて、主記憶装置からデータを取り出す。
    • 命令を実行する。
    • 実行された演算結果をレジスタ、または主記憶装置へ格納する。

※命令の実行は、命令の読み出し（命令フェッチ）　→　命令の解読（デコード）　→　有効アドレスの計算　→　オペランドの取り出し　→　命令の実行　→　結果の書き込みといったステージに分割できる。これで１ステージの実行に要する時間をマシンサイクルと呼ぶ。

一つの命令は、オペレータ部とオペランド部で構成されている。

  • オペレータ部：加算、減算、比較、分岐などの動作の種類を指定する。
  • オペランド部：アドレス情報。

オペランド個数による種類：

  • 0アドレス種類：スタックに…
  • 1アドレス種類：主記憶とcpu内
  • 2アドレス種類：主記憶
  • 3アドレス種類：主記憶

命令の種類

  • 算術演算命令
  • 論理演算命令
  • 転送命令
  • 比較命令
  • 分岐命令
  • シフト命令
  • 入出力命令

アドレス指定方式

  • 即値アドレス指定
  • 直接アドレス指定
  • 間接アドレス指定
  • 指標アドレス指定（インデックス修飾）

プロセッサ

クロックは、マザーボード上にある複数の装置が、タイミングを合わせて動作するために発生させる信号のこととし、クロック周波数とは、その信号の周波数を表す。
CPUは、基本的にはクロック周波数が高くなるほど高速に動作することができる。

クロック周波数の単位は、MHz（メガヘルツ）、またはGHz（ギガヘルツ）が使われる。

  • CPI（cycles per Instruction）は、一命令実行に必要なクロック数を表す単位である。
  • MIPS（million instructions per second）は、一秒間に実行可能な命令数の単位である。
  • FLOPS（floating-point operations per second）:一秒間に実行可能な命令数である。
  • 命令ミックス：CPUの処理性能を測定する単位である。

CPUを高速化する技術は、パイプライン処理がある。
この処理は、命令の読み出しから完了までのステージを独立して動作させることで、複数の命令を並行して実行する方式である。

  • スーパーパイプライン：パイプライン細分化することで、高速化を図る方式である。
  • スーパースカラ：一つのCPU内に複数の演算ユニットを内蔵し、複数のパイプラインで並列動作させる方式。
  • VLIW（very long instruction word）:長くとった命令語に複数の命令をまとめておき、並列動作させる方式。
  • マルチコアプロセッサ：一つのCPUの中に複数コアをもたせたもので、複数のCPUとして認識される。

メモリ

主記憶装置には、性質の違いでRAMとROMに分かれている。
RAMは読み込みと書き込み可能で、電源を切ると記憶情報が失われる（揮発性）。
ROMは基本的に読み込みだけ可能で、電源を切っても記憶情報は保持される（不揮発性）。

RAM（random access memory）は、次のような種類がある。

  • DRAM：大容量のものを比較的に安価に製造でき、主記憶装置に使用される。リフレッシュ動作が必要とする。
  • SRAM：フリップフロップ回路で構成され、回路が複雑で大容量化は困難だが、読み書きが高速でレジスタキャッシュメモリに利用される。リフレッシュは不要。

ROM（read only memory）は、次のような種類がある。

  • マスクROM：出荷時にデータが書き込まれており、内容の変更はできない。
  • PROM（programmable ROM）:あとから書き込みが可能なROM。ユーザーが一度書き込むことができるが、一度書き込んだものは消去できない。
  • フラッシュメモリ：EEPROMを改良し、ブロック単位もしくは一括でデータを消去、大容量化、書き込みの高速化などを実現した。

メモリが誤り検出と誤り訂正機能がある。

  • パリティチェック（parity check）：パリティビットを含めて１の数が偶数になるようにする偶数パリティと奇数になるようにする奇数パリティがある。
  • ECC（Error Correcting Code）：メモリないのデータのエラーを検出し、自動的に訂正できる。信頼性を求められるシステムに利用されている。

記憶階層は：
速度や容量の異なる複数のメモリモジュールやディスク装置を階層的に組み合わせることで、全体的に効率よい記憶システムを実現する。
レジスタ→キャッシュメモリ（SRAM）→主記憶装置（DRAM）→ディスクキャッシュ→補助記憶装置

キャッシュメモリは、CPUの動作速度と主記憶装置のアクセス時間のギャップを埋める、高速かつ小容量のメモリである。
キャッシュメモリのヒット率は、呼び出したい情報がキャッシュメモリ上にない確率をNFP（not found probability）、逆数の１-NFPをヒット率と呼ぶ。
CPUの実行アクセス時間＝ヒット率Xキャッシュメモリアクセス時間+（１－ヒット率）X主記憶のアクセス時間

キャッシュメモリに記録されている情報は、最終的には主記憶装置の該当する箇所に書き戻さなければならない。

  • ライトスルー方式　CPU→キャッシュメモリ・主記憶装置
  • ライトバック方式　CPU→キャッシュメモリ→主記憶装置

メモリインタリーブは、連続下領域が読み書きされる場合が多いという性質を利用して、アクセス効率を向上させる方式。
主記憶装置を複数のバンクに分け、バンクごとに独立してアクセスできるようにハブを配置。
